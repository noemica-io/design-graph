Quickstart

Go from npm install to five rendered variants on a spatial canvas in under two minutes. This guide walks through the four commands you'll use 90% of the time — init, spawn, branch, and open.

Last updated Mar 14 2026 · v0.4.2 · ~3 min read

01 Install#

design-graph ships as a single CLI with an embedded viewer. Install it globally with your package manager of choice — there is no separate server or daemon to run.

npm
pnpm
bun
brew
$npm install -g design-graph copy

Verify with design-graph --version. Requires Node ≥ 18 or Bun ≥ 1.0.

02 Initialize a project#

A project is just a directory with a graph-data.js manifest and a pages/ folder. init scaffolds both, plus a minimal config and a sample brief you can edit later.

Terminal zsh
$ design-graph init my-project ✓ created my-project/ ✓ wrote .design-graph/config.json ✓ wrote brief.md # edit this first ✓ ready run cd my-project && design-graph spawn
my-project/ tree
my-project/ ├── .design-graph/ │ ├── config.json # model, parallelism, paths │ ├── cache/ # prompt + response cache │ └── runs.log ├── pages/ # generated variants land here │ └── (empty) ├── graph-data.js # { nodes: [], edges: [] } ├── brief.md # your design prompt / brand voice └── README.md

graph-data.js is a plain JS file so you can version it in git and hand-edit nodes without a CLI round-trip.

03 Spawn variants#

spawn fans out N background agents in parallel. Each one reads the brief independently — no shared context, no peeking at siblings — so their directions actually diverge. Each agent emits a self-contained HTML file into pages/ and appends itself to graph-data.js the moment it finishes.

Terminal · live output zsh
$ design-graph spawn "pricing page for a SaaS" --n 5 › seeded from brief.md · model claude-opus-4.7 · depth 0 › spawning 5 background agents · no shared context 01 editorial-pricing ████████████ ready 18.1s 02 slider-calculator ████████████ ready 21.4s 03 comparison-table ████████████ ready 23.9s 04 chat-to-quote ████████▒▒▒▒ writing… 05 single-tier ▒▒▒▒▒▒▒▒▒▒▒▒ queued › 3 of 5 variants written to ./pages/ › elapsed 24.1s · tokens 41.2k · ✓ no errors › canvas: http://localhost:3333/?project=my-project
Note Variants run in parallel — 5 variants take ~90s, not 5×20s. Wall-clock time is bounded by the slowest agent, not the sum. Scale --n up to 16 without a meaningful latency tax.

Pass --brief ./custom-brief.md to override the project brief, or --lens "jakob,hick" to nudge each agent toward different UX principles.

04 Open the canvas#

open boots a local viewer at localhost:3333 and focuses the current project. The server is static — it just serves the files in pages/ and watches graph-data.js for changes so new variants appear live.

Terminal zsh
$ design-graph open › serving ./ at http://localhost:3333 › watching graph-data.js · live mode on › ⌘-click a node to open its page · press / to chat-iterate
What you'll see: a zoomable spatial canvas with one card per variant. Drag to cluster. Click a card to open the page at 1:1 inside a modal. Select two and hit ⌘\\ for side-by-side compare. Branch any node with ⌘B to spawn children that inherit its direction.
